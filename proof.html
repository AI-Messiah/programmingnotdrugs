Proof of solve-ability

All equations are solvable as long as they can return a real number.

To solve an equation that cannot be solved with other methods. You first need to put all of the unknown terms on one side of the equation. Then make the side that has the unknowns into a function. Then for each of the instances you want to solve for define the minimum and maximum values that the instance must be between and then if the function is increasing or decreasing.

Here is an example in C like code of how to do this.

Take 2 ^ X + 3X = 16 then write:

double func(double inv){
return pow(2, inv) + 3 * inv;
}

double hig = 3;
double low = 2;
double X = low;
for (float hval = (hig - low) / 2;hval > pow(2, -48);hval /= 2) if (func(X + hval) <= 16) X += hval;

What this means is that hval is set to half of the maximum minus the minimum. Then the function is tested for X + hval to see the result is smaller or equal to 16. If it is smaller or equal to 16 then hval is added to X. Then hval is divided by 2 and the process is repeated until hval is smaller than a defined size.

The float data type is smaller in precision than the double type. This doesn't matter because of how floating point numbers work.

func(X + hval) <= 16 this is because the function is increasing if it were decreasing it would be func(X + hval) >= 16.

To find 2 values at once you would assume that Y is a variable that is only increasing or decreasing and as you would solve for X which one would be increasing and the other decreasing. You solve for both X values and update Y.
